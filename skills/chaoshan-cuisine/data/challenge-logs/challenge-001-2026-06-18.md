CH-01: 加权中位数数学定义错误 + 多权重乘数偏差放大
──────────────────────────────────────────────────
提出者: 数据科学家 📊 + 公平性审计员 ⚖️
涉及章节: 原则4算法流程4.a（第187-190行原版），加权评分公式
严重程度: P0

论据摘要:
- 数据科学家：当前实现将 rating×weight 后的值取中位数（[5×1.0, 4×0.5, 3×0.3]=[5.0,2.0,0.9]→median=2.0），
  正确做法是按评分排序累计权重：3(权重0.3),4(累计0.8),5(累计1.8)→weighted median=4。两者结果差异显著。
- 公平性审计员：多权重连乘（time_weight×reviewer_weight）使边缘群体被指数级降权。
  被标记的远郊评价者time_weight(0.5)×reviewer_weight(0.3)=0.15，主流群体1.0×1.2=1.2，权重差8倍。

辩论摘要:
- 第1轮：本地老饕担心正确加权中位数让近期高分评价统治结果；营销号指出max_weight_decay保护被利用
- 第2轮：数据科学家回应——增加加权Q1/Q3展示纵贯信息；max_weight_decay默认0.3并增加二次标记永久化
- 第3轮投票：5✅(数据科学家/游客/老饕/文化研究者/审计员) + 1⚠️(营销号) = 6/6同意 → Valid Bug（全票）

修复方案:
1. 将加权评分公式改为标准化加权中位数：按评分排序，累计权重达50%时对应的评分值
2. 增加加权四分位数（weighted_Q1, weighted_Q3）展示纵贯分布
3. 新增 max_weight_decay_ratio 参数（默认0.3），防止多权重连乘过度惩罚
4. 二次标记永久化：解除标记后再次被标记的，触发永久标记（权重0.1，不可解除）

影响范围: 所有店铺的 weighted_median 计算结果会改变（从错误算法恢复到正确值）
回滚方案: 恢复原版第4.a步的加权评分公式 + 删除 max_weight_decay_ratio 参数

CH-02: 时间衰减对本地老饕不公 + 老店例外规则形同虚设
────────────────────────────────────────────────────
提出者: 本地老饕 🧓
涉及章节: 原则4（时间衰减），参数可调表 time_weights
严重程度: P0

论据摘要:
- >3年评价权重仅0.3，老店例外需关键词匹配"水平稳定""和以前一样"——过于机械
- 本地人常用表述如"味道还是那个味""三十年不变""火候一直稳"不被识别
- 非老店标签的隐世老铺（经营二三十年但没挂"老店"招牌）被全面牺牲
- 纵贯对比信息（老板换人、食材降级）恰恰最有价值却被最低权重

辩论摘要:
- 第1轮：游客认为时间衰减有其合理性；数据科学家建议连续指数衰减+动态λ；审计员担心纵贯加成只惠及本地人
- 第2轮：老饕接受连续衰减方案；将纵贯加成从+0.5降为+0.2；保留语义匹配核心诉求
- 第3轮投票：4✅(老饕/数据科学家/文化研究者/审计员) + 2⚠️(游客/营销号) = 6/6同意 → Valid Bug

修复方案:
1. 离散阶梯衰减 → 连续指数衰减函数（time_weight = exp(-λ × years)）
2. 按店铺类型动态λ值：网红/连锁店0.7，一般店铺0.3，稳定老店0.15
3. 关键词匹配 → 同义语义簇匹配（涵盖10+种本地常用表述）
4. 新增纵贯对比评价加权（+0.2权重加成）

影响范围: 所有店铺的 time_weight 计算结果改变（从离散阶梯到连续指数）
回滚方案: 恢复原版离散阶梯权重 + 关键词匹配规则

CH-06: 反操控检测参数组合系统性漏洞 + 多攻击向量
──────────────────────────────────────────────────
提出者: 数据科学家 📊 + 营销号操盘手 🎭
涉及章节: 原则6（反操控检测），suspicious_window_days=7, suspicious_min_reviews=3
严重程度: P0

论据摘要:
- 数据科学家：AND逻辑三条件——打破任意一个即绕过。方差阈值0.3在离散评分上存在真空区
- 营销号M-1"低空滑翔"：10账号每8天发2条评分[5,4]→永不触发，年产出520条零检测
- 营销号M-2"木马评级"：[5,5,3]方差0.89>0.3绕过，10人先积累可信权重再轮换攻击
- 标记不删除使F>3.33×R时虚假评价有效影响力超真实；空数据库首条评价无对照基线

辩论摘要:
- 第1轮：本地老饕质疑10账号协作成本；游客担心基线偏离误伤真实评价；审计员提醒隐私平衡
- 第2轮：数据科学家接受基线偏离方案替代硬阈值；行为模式分析从IP/设备改为浏览器指纹（非隐私方式）
- 第3轮投票：5✅(老饕/游客/数据科学家/营销号/审计员) + 1—(文化研究者弃权) = 5/5同意 → Valid Bug

修复方案:
1. AND逻辑 → 任意2条条件满足即触发（提高覆盖度）
2. 固定7天窗口 → 步长1天重叠式滑动窗口（消除边界规避）
3. 硬阈值 <0.3 → 基线偏离检测（window_variance < baseline × 0.5）
4. 硬阈值 >60% → 基线偏离检测（文本相似度 > 基线 + 2σ）
5. 新增第2层行为模式检测（提交间隔分析 + 小团体重合度检测）
6. 新增第3层二次标记永久化（权重0.1，不可解除）+ 申诉通道
7. min_reviewer_weight 从0.1提至0.3（第1层标记）

影响范围: 反操控检测灵敏度提高，更多之前可绕过的模式被捕获
回滚方案: 恢复原版AND逻辑 + 固定7天窗口 + 硬阈值参数

  - 新增参数：longitudinal_bonus(0.2), old_shop_semantic_cluster(true)

  - 新：三层系统——滑动窗口+基线偏离检测+行为模式分析+二次标记永久化
  - 旧参数移除：suspicious_similarity(0.60)
  - 新参数：suspicious_condition_trigger(2), variance_baseline_ratio(0.5), similarity_baseline_sigma(2)
