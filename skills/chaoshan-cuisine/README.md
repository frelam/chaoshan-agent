## 核心理念

1. **好吃是唯一标准** — 不看装修、不看名气、不看米其林星
2. **不做榜单，因人而异** — 汕头本地人、北京游客、广州回访客，每个人的"好吃"定义不同
3. **去营销化** — 不说"天花板""必打卡""绝绝子"，真实评价原文比营销话术贵一万倍
4. **声音多元** — 本地人和游客的口味差异本身就是信息，不是噪音
5. **算法公开透明** — 中位数抗极值、少数声音保留、反操控检测，全部开源可验证
6. **持续自演进** — 每周自动收集数据 + 多Agent挑战公平算法，不断迭代优化

## 为什么需要这个

| 平台 | 根本问题 |
|------|---------|
| 大众点评 | 刷分产业链成熟，评分被营销预算而非好吃程度决定 |
| 小红书 | KOL 流量垄断，普通食客声音被淹没，"天花板""绝绝子"式话术让信息失真 |
| 米其林/黑珍珠 | 评审匿名但不透明，法餐体系偏好无法代表潮汕本地口味 |
| 网红探店 | 本质是广告——商家付费 → 博主吹捧 → 粉丝踩雷 |

**本项目的答案：** 零商业收入 + 开源透明 + 7层反操控 + 人群分层评价 + AI驱动自演进。

## 7 层公平保障

### 第 1 层：资金来源——零商业利益

```
❌ 不接受商家付费推广
❌ 不接受免费餐食换取好评
❌ 不接受"霸王餐"式的评价交换
✅ 所有评价来自自费消费的真实食客
```

当榜单没有利益关联，商家就没有动力来操控它。

### 第 2 层：所有权——开源透明

全部数据公开在 GitHub——任何人可以查看每一条原始评价，检查算法是否正确执行，所有修改有 git 历史可追溯。

### 第 3 层：中位数代替平均数

```
例：8条真实评价 [5, 5, 5, 5, 5, 5, 5, 1]
  平均数 = 4.5  ← 被1条1分拉低
  中位数 = 5.0  ← 不受影响，反映真实水平
```

中位数天然抵抗极值操控。刷 10 个 5 分也好，刷 10 个 1 分也好，都动不了中位数。

### 第 4 层：少数声音保留

任何 ≥15% 评价者持有的观点都独立列出。

```
✅ 共识 (83%)：汤底鲜美、牛肉新鲜
💬 少数声音 (25%)：3/12 人觉得"太咸"（全部是游客）
```

对游客来说，"太咸"这条信息可能比"汤底鲜美"更有价值。

### 第 5 层：画像分层

不展示单一排名，展示多维度数据：

```
本地人评分: 5.0 ｜ 游客评分: 4.0 ｜ 回访客评分: 4.5
18-25岁: 3.8 ｜ 26-35岁: 4.3 ｜ 36-50岁: 4.5
```

每种差异不是噪音——它是信息。你知道自己属于哪个群体，就能找到对自己更有参考价值的评分。

### 第 6 层：反操控检测

自动检测刷分模式（7天内≥3条评价 + 评分几乎一致 + 文本高度相似），标记为 `suspicious_cluster` 并降权，但不删除。

### 第 7 层：权重上限

不存在"大 V 一条评价顶十条"的情况。单条评价权重上限为 2.0，即使最资深贡献者也不超过新人的 2 倍。

### 对比

| 维度 | 大众点评 | 小红书 | 米其林 | **本系统** |
|------|---------|--------|--------|----------|
| 资金来源 | 商家广告+霸王餐 | KOL广告 | 轮胎公司营销 | **零商业收入** |
| 评分算法 | 闭源黑盒 | 无评分 | 匿名不透明 | **开源可验证** |
| 声音多元 | 高分淹没低分 | 流量向头部集中 | 单一评审体系 | **少数声音保留** |
| 操控风险 | 刷分产业链成熟 | KOL可购买 | 评审可影响 | **7层反操控机制** |
| 人群差异 | 不区分 | 不区分 | 不区分 | **本地人/游客分层** |
| 数据可查 | 不可导出 | 不可导出 | 不公开 | **GitHub全量开源** |

## 个性化推荐

不设固定排名，排序因人而异。

```
你的画像（家乡/年龄/口味偏好/预算）
  → 匹配相似评价者
  → 按相似度加权聚合
  → 生成"适合你"的推荐
  → 同时展示全貌（大众评分 + 各人群细分）
```

### 相似度权重

| 维度 | 权重 | 匹配规则 |
|------|------|---------|
| 家乡 | 0.3 | 同城=1.0，同省=0.5 |
| 年龄段 | 0.2 | 完全匹配=1.0，相邻=0.5 |
| 身份类型 | 0.2 | local/tourist/returning-visitor |
| 口味标签 | 0.2 | 标签交集比例 |
| 价格区间 | 0.1 | 区间重叠度 |

### 推荐示例

```
🎯 针对【北京/26-35岁/游客】的推荐

1. 🏠 八合里海记牛肉店
   和你相似的食客评分: 4.0/5（基于5条相似评价）
   大众评分: 4.3/5（全部评价）
   推荐理由: "和你一样来自北京的游客都觉得脖仁很惊艳"
   推荐菜: 脖仁 ⭐5.0 / 吊龙 ⭐4.0

👥 各人群怎么看：
   本地人 4.5 | 游客 4.0 | 回访客 4.3
   18-25岁 3.8 | 26-35岁 4.3 | 36-50岁 4.5

💡 跟你口味不同的人说：
   ⚠️ 胸口朥：喜欢油香的人给5分，怕油腻的人给2分
```

详见 [`docs/PERSONALIZATION-ALGORITHM.md`](docs/PERSONALIZATION-ALGORITHM.md)

## 多Agent算法挑战赛

每周四，6 个不同立场的 Agent 对公平算法发起挑战，辩论后达成共识，输出算法优化方案。

### 6 个挑战者

| Agent | 立场 | 攻击方向 |
|-------|------|---------|
| 🧓 **本地老饕** | "算法太迁就游客" | 时间衰减过于激进、本地人声音被稀释 |
| 🧳 **外地游客** | "算法偏袒本地人" | 可信度评估不合理、忽略可及性 |
| 📊 **数据科学家** | "统计方法有缺陷" | 样本量置信区间、冷启动、评分偏差 |
| 🎭 **营销号操盘手** | "我能刷分绕过你" | 反操控盲点、审查周期漏洞 |
| 📖 **文化研究者** | "你忽略了文化语境" | 跨品类评分不可比、品类差异被忽视 |
| ⚖️ **公平性审计员** | "你的细分粒度不公" | 系统性偏见、人群细分不均、透明度缺乏 |

### 4 阶段流程

```
第1阶段：挑战提案
  └─ 每个Agent独立阅读 FAIRNESS-ALGORITHM.md + 实际数据
  └─ 提出 1-2 个具体挑战（理论或数据驱动）

第2阶段：公开辩论（3轮）
  └─ 交叉质疑 → 提出者回应 → 最终陈述
  └─ P0→P1→P2 优先级排序，批处理

第3阶段：共识投票
  └─ 🔴 Valid Bug → 讨论修复方案
  └─ 🟡 Edge Case → 记录在案
  └─ 🟢 Not Valid → 驳回

第4阶段：算法更新
  └─ 修复方案转化为具体参数/规则变更
  └─ 更新 FAIRNESS-ALGORITHM.md
  └─ 写入挑战日志 → rsync → git push
```

详见 [`cuisine-self-evolve/SKILL.md`](cuisine-self-evolve/SKILL.md) 第三节。

## 如何贡献评价

### 🟢 入门（10分钟）

1. Fork 本仓库
2. 打开 `data/restaurants.yaml`
3. 复制模板，填写你的评价
4. 提交 PR

### 评价要求

| ✅ 好的评价 | ❌ 不好的评价 |
|------------|-------------|
| "脖仁入口即化，油脂分布均匀，38元值这个价" | "超级好吃！天花板！" |
| "胸口朥涮4分钟后变脆爽，完全不腻" | "好吃好吃好吃" |
| "我是广州人，第一次吃潮汕牛肉火锅" | "必须打卡" |
| "扣1分因为排队40分钟" | 纯表情包 |

### 🔴 深度参与

- **自演进贡献**：改进数据收集策略（`cuisine-self-evolve/data/search-sources.md`）
- **算法挑战**：为每周四的挑战赛出新的攻击方向（`cuisine-self-evolve/SKILL.md`）
- **算法优化**：改进公平算法（`docs/FAIRNESS-ALGORITHM.md`）
- **牛肉知识**：补充牛肉部位知识（`data/beef-knowledge.yaml`）
- **新分类**：新增卤鹅、肠粉、甜汤等其他美食品类

详见 [`docs/CONTRIBUTION-GUIDE.md`](docs/CONTRIBUTION-GUIDE.md)

## 公平算法一览

| 机制 | 做什么 | 为什么 |
|------|--------|--------|
| 中位数评分 | 用 median 而非 mean | 防极端评分扭曲 |
| 少数声音保留 | ≥15%的观点独立列出 | 不让 70% 吃掉 30% |
| 人群分层 | 本地人/游客/年龄分层展示 | 口味差异是信息不是噪音 |
| 时间衰减 | 越老的评价权重越低 | 反映店铺现状 |
| 反操控 | 检测刷分集群，标记降权 | 标记而非删除 |
| 争议透明 | 评分方差>1.5 时标注 | 评价两极分化本身值得注意 |
| 多Agent挑战赛 | 每周6个角色挑战算法 | 持续发现盲点，自我优化 |

详见 [`docs/FAIRNESS-ALGORITHM.md`](docs/FAIRNESS-ALGORITHM.md)
