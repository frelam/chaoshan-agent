# 潮汕美食 (Chaoshan Cuisine) Skill — v2.0.0

> 不做榜单，不做排名。以好吃为准、去营销化的潮汕美食公平评价与个性化推荐体系。

## 这是什么

`chaoshan-cuisine` 是 [chaoshan-agent](https://github.com/frelam/chaoshan-agent) 项目的第二个技能（Skill）。它提供：

- 🍲 **真实食客评价库** — 每一条评价来自真实贡献者，好吃说好吃，不好吃说不好吃
- 📊 **公平聚合算法** — 用中位数抗极值、少数声音保留、反操控检测，确保总结不偏不倚
- 🥩 **潮汕牛肉知识库** — 12 个经典部位：口感、涮法、人群画像、全球命名对照
- 👤 **评价者画像系统** — 记录每位评价者的口味偏好、可信度和历史，支撑个性化推荐
- 🎯 **个性化推荐** — 根据"你是谁"（家乡、年龄、口味）给出因人而异的建议
- 🤖 **AI 美食导航** — 基于真实评价做推荐，不夹带营销私货

## 为什么我们的榜单是公平的

现有的美食榜单，无论大众点评、小红书还是米其林，都存在系统性的不公平：

| 平台 | 根本问题 |
|------|---------|
| 大众点评 | 刷分产业链成熟，商家买好评、买霸王餐换好评。评分被营销预算而非好吃程度决定。 |
| 小红书 | KOL 经济——流量向头部博主集中，普通食客的声音被淹没。"天花板""绝绝子"式话术让信息失真。 |
| 米其林/黑珍珠 | 评审匿名但不透明。评审员的个人口味偏好（通常是法餐体系的）决定一切，本地人的"从小吃到大"的口味标准不在评价维度里。 |
| 网红探店 | 本质是广告。商家付费 → 博主吹捧 → 粉丝踩雷。 |

### 我们的解决方案：7 层公平保障

#### 第 1 层：资金来源——零商业利益

```
本榜单不接受任何形式的商业合作：
  ❌ 不接受商家付费推广
  ❌ 不接受免费餐食换取好评
  ❌ 不接受"霸王餐"式的评价交换
  ✅ 所有评价来自自费消费的真实食客
  ✅ 贡献者动机：分享真正好吃的东西，而不是赚钱
```

**这是最根本的保障**。当一个榜单没有利益关联，商家就没有动力来操控它。

#### 第 2 层：所有权——开源透明

```
全部数据公开在 GitHub：
  ✅ 任何人都可以查看每一条原始评价
  ✅ 任何人都可以检查算法是否正确执行
  ✅ 任何人都可以 fork 数据、独立验证
  ✅ 任何修改都有 git 历史可追溯
```

大众点评的评分算法是闭源的黑盒。你看到一个 4.5 分的店，但不知道这个分数是怎么算出来的——是剔除了异常值？还是给大 V 更高权重？开源意味着**没有秘密**，公平是可验证的。

#### 第 3 层：中位数代替平均数

```
问题：1个恶意1分（或1个刷出来的5分）就能扭曲平均分
解法：用中位数 (median)

例：8条真实评价 [5, 5, 5, 5, 5, 5, 5, 1]
  平均数 = 4.5  ← 被1条1分拉低
  中位数 = 5.0  ← 不受影响，反映真实水平
```

中位数天然抵抗极值操控。你刷 10 个 5 分也好，竞争对手刷 10 个 1 分也好，都动不了中位数。

#### 第 4 层：少数声音保留

```
问题：如果 70% 的人说"好吃"，30% 的人说"太咸"，
      传统聚合会忽略"太咸"的声音，只展示"好吃"
解法：任何 ≥15% 评价者持有的观点都独立列出

例：
  ✅ 共识 (83%)：汤底鲜美、牛肉新鲜
  💬 少数声音 (25%)：3/12 人觉得"太咸"（全部是游客）
```

**对于来自外地的游客，这条"太咸"的信息比"汤底鲜美"更有价值。** 少数声音不被吞噬，反而被保留——因为对另一群人来说它可能是最重要的一条信息。

#### 第 5 层：画像分层

```
传统榜单：一家店只有一个分数
本榜单：  本地人评分 5.0 ｜ 游客评分 4.0 ｜ 回头客评分 4.5

为什么？因为本地人和游客的口味基准不同：
  - 本地人：以"从小吃到大"为标准，偏咸偏鲜都觉得正常
  - 游客：  以自己家乡的口味为基准，可能觉得偏咸/偏淡/偏油
  - 回头客：介于两者之间，有跨城市对比的经验

这种差异不是噪音——它是信息。你知道自己属于哪个群体，
就能找到对自己更有参考价值的评分。
```

#### 第 6 层：反操控检测

```
自动检测刷分模式：
  🚩 同一家店 7 天内出现 ≥3 条评价
  🚩 这些评价的评分几乎完全一致（方差 < 0.3）
  🚩 评价文本高度相似（关键词重叠 > 60%）

检测到 → 标记为 suspicious_cluster
  - 原始评价保留（不删除）
  - 聚合计算时降权（权重 × 0.3）
  - 总结中标明"含可疑评价"
```

⚠️ 可能误判（比如一群朋友同一天去同一家店，各自写了相似的评价），所以只标记不删除。如果这些评价者后续提交了其他店铺的有差异的评价，自动解除标记。

#### 第 7 层：每个人的话语权上限

```
新用户初始权重: 1.0
贡献 ≥5 条有实质内容的评价: 1.2 (小幅提升)
被标记可疑的评价者:      0.3 (大幅降低)

单条评价的权重上限: 2.0 ← 这是安全阀

即使是最资深的贡献者，权重也不超过新人的 2 倍。
不存在"大 V 一条评价顶十条"的情况。
```

这 7 层保障环环相扣：**没有钱 → 没有刷分动机 → 算法防操控 → 少数声音保留 → 人群分层展示 → 全部开源可验证**。这不是一个"我们觉得公平"的榜单——公平性是设计到系统里的，可以被任何人验证。

### 与现有平台的对比

| 维度 | 大众点评 | 小红书 | 米其林 | **本榜单** |
|------|---------|--------|--------|----------|
| 资金来源 | 商家广告+霸王餐 | KOL广告 | 轮胎公司营销 | **零商业收入** |
| 评分算法 | 闭源黑盒 | 无评分 | 评审匿名不透明 | **开源可验证** |
| 声音多元 | 高分淹没低分 | 流量向头部集中 | 单一评审体系 | **少数声音保留** |
| 操控风险 | 刷分产业链成熟 | KOL可购买 | 评审可影响 | **7层反操控机制** |
| 人群差异 | 不区分 | 不区分 | 不区分 | **本地人/游客分层** |
| 数据可查 | 不可导出 | 不可导出 | 不公开 | **GitHub全量开源** |

## 快速开始

### 安装

```bash
# 克隆仓库
git clone https://github.com/frelam/chaoshan-agent.git
cd chaoshan-agent

# 安装到 Hermes Agent
bash install.sh chaoshan-cuisine
```

### 使用

在 Hermes Agent 中：

```
> 汕头哪家牛肉火锅好吃？
> 推荐吃的 — 我是北京来的，第一次来汕头
> 潮汕攻略 — 揭阳有哪些必吃美食？
> 脖仁是什么部位？和吊龙有什么区别？
> 胸口朥在日本叫什么？
```

## 如何贡献评价

### 🟢 入门（10分钟）

1. Fork 本仓库
2. 打开 `data/restaurants.yaml`
3. 复制文件底部的 TEMPLATE 模板
4. 填写你的真实评价（必填字段：店名、地址、点了什么菜、好吃程度、为什么）
5. 提交 PR

### 评价要求

| ✅ 好的评价 | ❌ 不好的评价 |
|------------|-------------|
| "脖仁入口即化，油脂分布均匀" | "超级好吃！天花板！" |
| "胸口朥涮4分钟后变脆爽，完全不腻" | "好吃好吃好吃" |
| "扣1分因为排队40分钟" | "满分满分满分" |
| "我是广州人，第一次吃潮汕牛肉火锅" | "必须打卡" |

- 改进公平算法（`docs/FAIRNESS-ALGORITHM.md`）

## 设计原则

| 原则 | 说明 |
|------|------|
| **好吃是唯一标准** | 不看装修、不看名气、不看米其林星 |
| **去营销化** | 不说"天花板""必打卡""绝绝子" |
| **声音多元** | 本地人和游客的口味差异本身就是信息 |
| **算法公平** | 中位数抗极值，少数声音保留，反操控检测 |
| **开源共建** | 所有人的评价权重起点相同，没有 KOL 特权 |

## 公平算法一览

| 机制 | 做什么 | 为什么 |
|------|--------|--------|
| 中位数评分 | 用 median 而非 mean | 防极端评分扭曲 |
| 少数声音保留 | ≥15%的观点独立列出 | 不让 70% 吃掉 30% |
| 人群分层 | 本地人/游客/回头客分开展示 | 口味差异是信息不是噪音 |
| 时间衰减 | 老评价降权 | 反映店铺现状 |
| 反操控 | 检测刷分集群 | 标记而非删除 |
| 争议透明 | 评分方差>1.5 时标注 | 评价两极分化本身值得注意 |

详见 [`docs/FAIRNESS-ALGORITHM.md`](docs/FAIRNESS-ALGORITHM.md)

## 个性化推荐

v2.0.0 全新能力：不设固定排名，排序因人而异。

```
你的画像（家乡/年龄/口味）
  → 匹配相似评价者
  → 按相似度加权聚合
  → 生成"适合你"的推荐
  → 同时展示全貌（大众评分 + 各人群细分）
```

| 维度 | 权重 | 说明 |
|------|------|------|
| 家乡 | 0.3 | 同城=1.0，同省=0.5 |
| 年龄段 | 0.2 | 完全匹配=1.0，相邻=0.5 |
| 身份类型 | 0.2 | 本地人/游客/回访客 |
| 口味标签 | 0.2 | 标签交集比例 |
| 价格区间 | 0.1 | 区间重叠度 |

详见 [`docs/PERSONALIZATION-ALGORITHM.md`](docs/PERSONALIZATION-ALGORITHM.md)
