# 潮汕美食 — 个性化推荐算法

> **不做榜单，不做排名。** 排序是针对具体用户的，因人而异。
>
> 这份文档定义如何根据"你是谁"来给出个性化建议。

---

## 核心原则

1. **排序不是固定的** — 同一家店对汕头本地人和北京游客的意义完全不同
2. **找和"你"相似的人** — 从评价数据库中找到画像相似的人群，看他们怎么说
3. **保留多元视角** — 同时展示"大众观点"和"和你相似的人的观点"
4. **透明解释** — 告诉用户为什么推荐这个（"像你这样的年轻人觉得..."）

---

## 输入：用户画像

### 收集方式

```
直接询问用户：
  1. 你从哪里来？          → hometown
  2. 你多大？              → age_range
  3. 你是一个人来吃         → visit_context
     还是带家人/朋友？
  4. 你有什么口味偏好？     → taste_tags
     喜欢油香的还是清淡的？
     喜欢弹牙的还是软嫩的？
  5. 预算如何？            → price_range
     人均50以内 / 50-100 / 100+
```

### 用户画像结构

```python
user_profile = {
    "hometown": "北京",          # 精确到城市
    "age_range": "26-35",        # 18-25 / 26-35 / 36-50 / 50+
    "type": "tourist",           # local / tourist
    "companions": "朋友",        # 独食 / 朋友 / 家人 / 情侣
    "declared_taste_tags": ["喜欢嫩滑", "怕油腻"],
    "price_range_lower": 30,
    "price_range_upper": 80,
    "preferred_cuisines": ["牛肉火锅", "肠粉"]  # 可选
}
```

## 核心算法

### Step 1: 找相似人群

从 `reviewer-profiles.yaml` 中找出和当前用户画像相似的评价者。

**相似度计算：**

| 维度 | 权重 | 匹配规则 |
|------|------|---------|
| hometown | 0.3 | 同一城市=1.0，同省=0.5，异地=0.2 |
| age_range | 0.2 | 完全匹配=1.0，相邻段=0.5 |
| type (local/tourist) | 0.2 | 完全匹配=1.0 |
| taste_tags | 0.2 | 匹配的标签数/总标签数 |
| price_range | 0.1 | 价格区间重叠度 |

```python
def similarity(user_a, user_b):
    score = 0
    # 家乡相似度
    if user_a.hometown == user_b.hometown:
        score += 0.3
    elif user_a.hometown_province == user_b.hometown_province:
        score += 0.15
    # 年龄段相似度
    if user_a.age_range == user_b.age_range:
        score += 0.2
    elif abs(user_a.age_index - user_b.age_index) == 1:
        score += 0.1
    # 身份类型相似度
    if user_a.type == user_b.type:
        score += 0.2
    # 口味偏好相似度
    shared_tags = len(set(user_a.taste_tags) & set(user_b.taste_tags))
    total_tags = len(set(user_a.taste_tags) | set(user_b.taste_tags))
    if total_tags > 0:
        score += 0.2 * (shared_tags / total_tags)
    # 价格偏好相似度
    score += 0.1 * price_overlap(user_a, user_b)
    return score
```

**阈值：** 相似度 > 0.5 视为"相似人群"。

### Step 2: 从相似人群的评价中提取信息

找到相似评价者后，聚合他们去过的店铺信息：

```python
similar_reviewers = find_similar(user_profile, threshold=0.5)

for reviewer in similar_reviewers:
    reviews = get_reviews_by(reviewer.id)
    for review in reviews:
        weight = similarity(user_profile, reviewer)
        # 时间衰减
        weight *= time_decay(review.visit_date)
        # 加入聚合
        restaurant_scores[review.restaurant_name] += review.overall_rating * weight
```

### Step 3: 生成推荐

**推荐结果包含两部分：**

```
推荐给 [用户] 的店铺（按相似人群评分排序）：

1. 🏠 八合里海记牛肉店
   和你相似的食客评分: 4.5/5（基于N条来自相似人群的评价）
   大众评分: 4.3/5（基于全部N条评价）
   推荐理由: "和你一样来自北京的游客都觉得这家脖仁很惊艳"
   推荐菜: 脖仁（雪花）⭐ 38元
   
2. 🏠 老姿娘蚝烙
   和你相似的食客评分: 4.0/5
   大众评分: 4.0/5
   推荐理由: "像你这样第一次来潮汕的人，蚝烙是很棒的入门选择"
   ...

同时展示"和你画像不同的人的看法"：
  💡 汕头本地人的看法：这家在本地人中评分 4.5/5
  💡 年轻人的看法（18-25岁）：评分 3.8/5
```

## 特殊场景处理

### 场景1：数据不足（新店/评价太少）

当某家店的评价数量 < 5 条：
- 不推荐使用个性化算法
- 直接展示所有原始评价原文
- 标注"评价样本较少，仅供参考"

### 场景2：口味相反的人群

如果用户"怕油腻"，但某家店的招牌恰好是"胸口朥"（纯脂肪）：
- 推荐指数打折扣（0.5x）
- 在推荐理由中明确标注："这家以胸口朥闻名，和你'怕油腻'的偏好可能不太匹配"
- 列出该店适合该用户的替代菜品

### 场景3：争议店铺

当 `restaurant-summary.yaml` 中 `controversy.is_controversial = true`：
- 在推荐中标注"争议店"标记
- 展示双方观点的原始评价原文
- 不做倾向性排序，让用户自己判断

### 场景4：首次使用（无用户历史）

用户未提供 taste_tags 时：
- 默认按 tourist 身份做推荐
- 优先推荐"首次来潮汕"的经典店和经典菜
- 在推荐后邀请用户补充偏好信息

## 算法局限说明

1. **相似度计算简化版** — 目前仅基于基本画像属性
2. **冷启动问题** — 新用户没有历史评价，只能靠显式偏好
3. **品味变化** — 用户的口味可能随时间变化，目前未建模
4. **评价饱和度** — 某家店评价太多会稀释个人权重

这些局限会在后续版本中逐步优化。

## 与公平算法的关系

```
个性化推荐算法                   公平展示算法
   │                                │
   ├─ 找出"和你相似的人"            ├─ 展示所有声音
   ├─ 加权聚合相似人群的评分         ├─ 保留少数派观点
   ├─ 生成个性化排序                 ├─ 不隐藏差评
   └─ 透明解释"为什么推荐这个"        └─ 反操控检测
   
两者互补，并行输出。
用户看到的是一份推荐 + 一份全貌。
```

## 未来改进方向

1. **评价文本情感分析** — 从文本自动提取 taste_tags
2. **协同过滤** — 用户-评价矩阵做协同过滤推荐
3. **渐进式交互** — 用户可以对推荐结果反馈（"这个推荐不对"→调整权重）
4. **季节性推荐** — 夏季推清淡，冬季推火锅
5. **随行人员建模** — "和朋友来" vs "带家人来"的不同需求
